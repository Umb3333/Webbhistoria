Googles switch from http to https

In 2018 Google announced that they would be marking all website that dont use Https as "unsafe".

Google's move to switch to HTTPS (Hypertext Transfer Protocol Secure) was not a sudden change in 2018; rather, it was part of a broader initiative that started several years earlier. HTTPS is the secure version of HTTP, which is the protocol used for transmitting data between a user's web browser and the website they are connected to. The "S" in HTTPS stands for "Secure," indicating that the communication is encrypted.

Google had been advocating for a more secure web for quite some time due to the increasing concerns about privacy, data integrity, and security on the internet. The transition to HTTPS was a gradual process, and Google took various steps to encourage website owners to adopt HTTPS:

  1. Ranking Signal: In 2014, Google announced that HTTPS would be used as a ranking signal in its search algorithm. This meant that websites using HTTPS would receive a slight boost in search engine rankings compared to non-secure websites. This move was intended to incentivize website owners to adopt secure connections.

  2. Browser Warnings: Google started marking HTTP sites as "Not Secure" in Chrome, its popular web browser, in 2017. This visual indication served as a warning to users that the connection to the website was not encrypted and their data might be at risk.

  3. July 2018 Deadline: In 2018, Google set a deadline for websites to migrate to HTTPS. Starting in July 2018, with the release of Chrome 68, all HTTP sites were explicitly marked as "Not Secure." This step was a part of Google's broader effort to make the internet a safer place by encouraging the use of encrypted connections.
By promoting the use of HTTPS, Google aimed to enhance user privacy, protect data integrity, and create a more secure online environment. Many other organizations and web browsers also supported and followed similar initiatives to promote a more secure and privacy-focused web.
